# Governance & Process (GOV)

**ID Prefix**: `GOV-###`
**Owner**: Engineering Lead
**Phase**: All phases

## Purpose

Consistency and maintainability through SDLC policy, coding standards, anti-patterns, and contribution guidelines.

## What Belongs Here

- **SDLC Policy**: Development lifecycle standards
- **Over-Engineering Guardrails**: Complexity management
- **Service Standards**: Service layer patterns and templates
- **Frontend Standards**: UI/UX component patterns
- **Hooks Standards**: React Query patterns
- **DTO Standards**: Type system patterns
- **Coding Standards**: Language-specific best practices
- **Contribution Guide**: PR process and review standards
- **Anti-Pattern Catalog**: What NOT to do and why
- **Pattern Library**: Reusable design patterns
