# AI-Native Resume Direction — Final Scaffold Precis

## Purpose
This document captures the corrected structural direction for the AI-native resume. It resolves ambiguity between:
- agentic workflows vs system design
- framework vs product
- experience vs projects

---

## Core Insight

The system consists of two distinct but coupled layers:

### 1. Agentic Delivery System (Liminal)
- Multi-agent orchestration layer
- Evaluation and validation harnesses
- HITL governance
- Memory and checkpoint persistence
- Tool integration and execution control

This aligns with industry understanding of agent orchestration as:
"coordinating multiple specialized agents to achieve shared objectives."

### 2. Application System (PT-2 / Cortalis)
- Conventional vertical SaaS
- DDD service architecture
- PostgreSQL RLS tenant isolation
- Transactional outbox
- Event-driven system

This is NOT an agentic system.

---

## System Relationship

Liminal (delivery system) → produces → PT-2 (SaaS)

The resume must communicate:
A system that builds reliable software, proven via a real product.

---

## Structural Rules

### Core Capabilities
Represents ONLY the agentic delivery system:
- Agentic Orchestration
- Evaluation Harness
- HITL Governance
- Compound Systems
- Observability & Fault Tolerance

### Experience
Single entry:
Liminal Tech — AI Engineer

Contains:
- PT-2 outcomes (WHAT)
- Powered by agentic system (HOW)

---

## Bullet Structure

Each bullet follows:

[PT-2 outcome]

## Constraints

- PT-2 must NOT be described as agentic
- No separate Projects section

## Mental Model

Core Capabilities = how software is built

---

## Final Positioning

Built an AI-native delivery system (Liminal) and validated it by producing a production-grade SaaS (PT-2) under real constraints.

## Verdict

This structure:
- removes ambiguity
- aligns with real agentic systems
- avoids AI hype
- preserves both system design and execution proof
