## 1. Executive Verdict

**`patch_required_before_PRD`**

The ADR set (059 · 060 · 061) is internally coherent. Two P1 semantic cracks require targeted patches before the PRD may proceed:

1. **CRACK-001 (P1):** ADR-060 D4 removes "Estimated Win/Loss" from allowed surface labels; UL Baseline §Surface rendering rules and Classification YAML `allowed_surface_labels` still permit it. These baseline artifacts need explicit alignment patches.
2. **CRACK-002 (P1):** ADR-059 D3 defines `source_authority.inventory`; ADR-060 D3 explicitly supersedes this with `source_authority.snapshots`. ADR-059 has not been inline-amended. Any implementor reading ADR-059 alone produces the wrong DTO shape. The TIA-CANON-SOURCE-AUTHORITY-SHAPE acceptance test fails against ADR-059 as written.

Three P2 gaps should be resolved concurrently but are not hard blockers if captured as explicit PRD acceptance criteria:

3. **CRACK-003 (P2):** Classification YAML classifies `estimated_drop_cents` as `acceptable_with_label`; ADR-060 D1 demotes it to `deprecated_alias — do not introduce in new code`.
4. **CRACK-004 (P2):** RATED_ADJUSTMENT is unaddressed in the UL Baseline and Classification YAML. Full exclusion semantics only appear in ADR-060 D2 and ADR-061 D2. If an implementor reads only baseline artifacts, RATED_ADJUSTMENT's exclusion is invisible.
5. **CRACK-005 (P2):** UL Baseline names the completeness field `input_completeness` (two-value: `complete`/`partial`). ADR-059 D3 nests it as `completeness.status` with three values: `'complete' | 'partial' | 'integrity_failure'`. Structural naming divergence.

The RATED_ADJUSTMENT exclusion rule is correctly implemented in ADR-060 D2 and ADR-061 D2. No artifact falsely claims RATED_ADJUSTMENT is schema-absent or structurally blocked. The exclusion is semantic/canonical, not structural — consistent with the TRIAGE special rule. No P0 RATED_ADJUSTMENT defect.

No P0 cracks exist within the ADR set itself. The P0 dashboard-suppression gate (Classification YAML `prd_gate_patches.dashboard_suppression_gate`) is open and must be absorbed into the exemplar PRD acceptance criteria.

### TIA-CANON-RATED-ADJUSTMENT-EXCLUSION

**Authority:** ADR-060 D2, ADR-061 D2

Setup:
- Create a table session with valid opener/closer/fill/credit inputs.
- Insert within-session-window `table_buyin_telemetry` rows:
  - `RATED_BUYIN` `amount_cents = 10000`
  - `GRIND_BUYIN` `amount_cents = 5000`
  - `RATED_ADJUSTMENT` `amount_cents = 999999`

- `telemetry_derived_drop_estimate_cents = 15000`
- `drop_estimate_state = 'present'`
- `calculation_kind = 'telemetry_drop_formula'`
- `projected_table_win_loss_cents` uses 15000 only (not 1014999)
- `RATED_ADJUSTMENT` has zero effect on canonical projection, `drop_estimate_state`, or any DTO field
- `completeness.status = 'complete'`
- `missing_inputs = []`

### TIA-CANON-NULL-VS-ZERO

**Authority:** ADR-059 D2, ADR-061 D6

Setup:
- **Case A:** No `RATED_BUYIN` or `GRIND_BUYIN` rows within the session window.
- **Case B:** `RATED_BUYIN` and `GRIND_BUYIN` rows exist within window; all have `amount_cents = 0`.

Expected:
- **Case A:** `telemetry_derived_drop_estimate_cents = null`; `drop_estimate_state = 'none_for_session'`; `calculation_kind = 'inventory_only'`; `projected_table_win_loss_cents = null`; `partial_table_result_cents` set.
- **Case B:** `telemetry_derived_drop_estimate_cents = 0`; `drop_estimate_state = 'present'`; `calculation_kind = 'telemetry_drop_formula'`; `projected_table_win_loss_cents` set (may be negative).
- In both cases: no `COALESCE` path converts null to 0 or 0 to null anywhere in `TableInventoryAccounting`.

## 6. Final Recommendation

### ADR set may proceed to PRD? **Conditionally yes — two patches required first.**

The ADR set (059 · 060 · 061) is substantively sound. The session-scope boundary is frozen. The RATED_ADJUSTMENT exclusion is correctly handled at the ADR layer. The three-result-state model is coherent. The null-vs-zero invariant is explicit and consistent. No artifact makes a false structural claim about RATED_ADJUSTMENT's absence.

**Patches required before PRD authorization:**

#### Patch 1 (P1 — blocking): Remove "Estimated Win/Loss" from UL Baseline and Classification YAML

- **UL Baseline §Surface rendering rules — projected win/loss state block:** Delete "Estimated Win/Loss: $X ← acceptable alternative label". Add: "`Estimated Win/Loss` is forbidden — ADR-060 D4 removed it from the allowed list as ambiguous. Use `Projected Win/Loss` only."
- **Classification YAML `allowed_surface_labels`:** Remove `"Estimated Win/Loss"` entry. Add explicit `forbidden_surface_labels` amendment noting ADR-060 D4.

#### Patch 2 (P1 — blocking): Inline-amend ADR-059 D3 `source_authority` block

- In ADR-059 D3, the `source_authority` TypeScript interface block: replace `inventory: 'table_inventory_snapshot'` with `snapshots: 'table_inventory_snapshot'`. Add amendment note inline: `// KEY AMENDED: 'inventory' → 'snapshots' per ADR-060 D3.`
- Add a header note to ADR-059 D3 section: "D3 `source_authority` shape is amended by ADR-060 D3. The `inventory` key no longer exists; the authoritative key is `snapshots`. ADR-060 D3 is the governing definition."

**Patches recommended before PRD (P2 — not blocking if captured as PRD acceptance criteria):**

#### Patch 3 (P2): Classification YAML `estimated_drop_cents` status

- Change `status: acceptable_with_label` → `status: deprecated_alias_transitional`. Add note: "Forbidden at DTO/API/surface boundary per ADR-060 D1/D4."

#### Patch 4 (P2): UL Baseline and Classification YAML — add RATED_ADJUSTMENT exclusion

- UL Baseline canonical drop taxonomy table: add `RATED_ADJUSTMENT` row with status "non-canonical / excluded" and a note consistent with ADR-060 D2.
- Scaffold §4 telemetry predicate: add inline comment: `-- RATED_ADJUSTMENT intentionally excluded per ADR-060 D2; may exist in legacy paths but must not contribute to canonical SUM`.

#### Patch 5 (P2): UL Baseline completeness envelope table

- Rename `input_completeness` → `completeness.status` to match ADR-059 D3 nesting.
- Add `integrity_failure` as third allowed value.

**Gate item open — must be absorbed into PRD:**
