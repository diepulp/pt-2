# Product Requirements (PRD)

**ID Prefix**: `PRD-###`
**Owner**: Product
**Phase**: Inception, Discovery, Design, Test, Release, Evolve

## Purpose

Drives prioritization and testable outcomes through user stories, acceptance criteria, constraints, and release goals.

## What Belongs Here

- **Product Requirements Documents**: Feature specifications with user stories
- **Release Plans**: Roadmap and phased delivery strategy
- **Feature Specs**: Detailed functional requirements
- **User Flows**: Journey maps and interaction patterns
- **Acceptance Criteria**: Testable outcomes for features
- **MVP Scope**: Minimal viable product definition

## Document Format

```yaml
---
id: PRD-001
title: Player Management System Requirements
owner: Product
status: Accepted
affects: [ARCH-012, API-005, QA-008]
created: 2025-10-25
last_review: 2025-10-25
---
```
