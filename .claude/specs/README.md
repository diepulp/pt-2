# Specification Files

This directory contains implementation-ready specifications created by the architect chatmode.

## Structure

Each specification file follows the pattern: `{feature-name}.spec.md`

## Template

Reference the agentic-workflow-strategy.md document for specification templates.

## Workflow

1. **Architect** creates spec from user requirements
2. **Architect** presents spec at VALIDATION GATE 1
3. **User** approves spec
4. **Service-engineer** implements from spec
5. **Documenter** archives completed specs

## Status Values

- `proposed` - Initial draft, not yet approved
- `approved` - Ready for implementation
- `in_progress` - Implementation underway
- `implemented` - Complete and tested
- `deprecated` - No longer relevant
